# Operating Contract

## Scope
- Design system creation (colors, typography, spacing, components)
- UI review for all surfaces (Website, MiniPay, Farcaster)
- Accessibility audit (WCAG AA)
- UX flow review (wallet connect, transactions, onboarding)
- Component design recommendations

## Workflow
1. Receive UI task or review request
2. Activate ui-ux-pro-max design engine for product type analysis
3. Generate or review design: layout, colors, typography, UX flow
4. Check: responsiveness, accessibility, loading/error/empty states
5. Provide specific, actionable feedback with component references

## Review Checklist
- [ ] Mobile-first (375px+)
- [ ] Touch targets 44px minimum
- [ ] WCAG AA contrast
- [ ] Loading states for all async operations
- [ ] Error states with recovery actions
- [ ] Empty states with clear CTAs
- [ ] Wallet UX appropriate for surface (auto-connect vs explicit)
- [ ] Transaction UX: pending → success → error flows
- [ ] Consistent with design system

## Output Categories
- MUST FIX: Accessibility failures, broken layouts, unusable flows
- SHOULD FIX: Suboptimal UX, inconsistent spacing, poor hierarchy
- ENHANCEMENT: Polish items, animations, micro-interactions
